# Examiner Victoria V3 国内移动端与微信人工测试清单

本清单用于 V3 国内 H5 公测前的人工验证。测试时不得记录真实用户姓名、私密语音、敏感回答、Cookie、Token 或真实 `.env` 值。

## 测试分类

- 自动检测：系统检测面板可直接显示的环境能力。
- 人工测试：需要测试者手动操作页面。
- 需要真实 API：会触发 STT、LLM、TTS 或报告生成的真实链路。
- 不需要真实 API：只验证页面、浏览器能力、本地播放、本地存储或复制能力。

## 手工步骤

1. 打开页面，确认首屏无白屏，主练习界面可见。
2. 打开 `System check`，确认页面环境、API、录音能力、本地播放、localStorage、下载/复制能力均有结果。
3. 在微信内置浏览器中确认出现 Safari / Chrome 建议，不直接阻止继续。
4. 点击“测试麦克风权限”，确认授权弹窗或失败提示清晰，测试结束后麦克风不持续占用。
5. 点击“测试声音播放”，确认只有用户点击后播放本地声音。
6. 点击“复制诊断结果”，确认剪贴板内容不包含 Cookie、Token、API Key、语音、回答或练习记录。
7. 开始一次最短练习，测试录音开始和停止。
8. 上传音频并观察 STT 是否返回转写；该步骤需要真实 API。
9. 提交答案并观察 LLM 回复；该步骤需要真实 API。
10. 测试 TTS 播放；如失败，应继续显示文字题目或反馈，并提示语音暂不可用。
11. 生成报告；该步骤需要真实 API。
12. 测试导出或下载；微信内若失败，使用复制结果作为降级。
13. 切到后台再返回，确认页面未卡死，录音状态和播放状态可恢复或给出提示。

## 记录格式

每次测试建议记录：设备、系统版本、浏览器、网络、是否微信内置浏览器、系统检测复制文本、失败步骤、是否可通过 Safari/Chrome 规避。不要记录真实回答全文或音频文件。
